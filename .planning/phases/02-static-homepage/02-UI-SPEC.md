## Typography

All typography uses DINNextW1G. Perfect Fourth ratio (1.333) modular scale already established in Phase 1.

### Type Scale

| Role | CSS Variable | Computed Size (base 16px) | Weight | Line Height | Usage in Phase 2 |
|------|-------------|---------------------------|--------|-------------|-------------------|
| Small / Label | `--type-sm` | 12px (0.75rem) | Regular 400 | `--leading-normal` (1.4) | Progress counter, copyright, meta dates |
| Body | `--type-md` | 16px (1rem) | Regular 400 | `--leading-normal` (1.4) | Nav items (D-07), footer body text (D-39), card descriptions, visit info |
| Section Label | `--type-md` | 16px (1rem) | Bold 700 | `--leading-tight` (1.1) | Footer section headers: uppercase + letter-spacing (D-39) |
| Card Title | `--type-lg` | 21px (1.333rem) | Bold 700 | `--leading-tight` (1.1) | Exhibition card titles, offering titles |
| Section Heading | `--type-2xl` | 38px (2.369rem) | Bold 700 | `--leading-tight` (1.1) | Section headings (Exhibitions, Visit, etc.) |
| Display | `--type-3xl` | 51px (3.157rem) | Bold 700 | `--leading-tight` (1.1) | Footer marquee text (D-36) |
| Hero Display | `--type-4xl` | 67px (4.209rem) | Bold 700 | `--leading-tight` (1.1) | "KUNST / HAUS / ZURICH" hero typography (D-01) |

### Hero Typography Specifics (D-01)
- Text: "KUNST / HAUS / ZURICH" -- three lines, right-aligned
- Font: DINNextW1G Bold 700
- Size: `clamp(var(--type-3xl), 8vw, var(--type-4xl))` -- scales with viewport, floor at ~51px, ceiling at ~67px
- Line height: `--leading-tight` (1.1)
- Z-index: topmost layer (above hero images per D-01)
- Text transform: uppercase
- Color: `var(--color-text)`

### Footer Section Headers (D-39)
- Font: DINNextW1G Bold 700
- Size: 16px (`--type-md`)
- Text transform: uppercase
- Letter spacing: 0.05em
- Color: `var(--color-text)`

### Nav Typography (D-07)
- Font: DINNextW1G Regular 400
- Size: 16px (`--type-md`)
- Line height: `--leading-normal` (1.4)

Source: Phase 1 CONTEXT.md D-07/D-08, existing `src/styles/typography.css`, Phase 2 CONTEXT.md D-01/D-07/D-39

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fbf8f7` (off-white) | Page background, hero background, most section backgrounds |
| Secondary (30%) | `#EBEBEB` (concrete-light) | Nav bar background (D-07), alternate section backgrounds, cards |
| Accent (10%) | `#272523` (soft-black) | All text, CTA button fills, footer dark bar, links |
| Decorative | `#C1BFB2` (concrete-warm) | Decorative borders, progress arc, dividers |
| Decorative Shadow | `#A4A197` (concrete-shadow) | Decorative only -- never for text (fails AA contrast) |

### On-site Mode -- from existing `semantic.css`

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#272523` (soft-black) | Page background |
| Secondary (30%) | `#A4A197` (concrete-shadow) | Decorative surfaces, card borders |
| Accent (10%) | `#fbf8f7` (off-white) | All text, CTA button fills, links |
| Decorative | `#C1BFB2` (concrete-warm) | Decorative elements (same in both modes) |

### Section Background Rotation (D-26)

4-step cycle repeating through homepage sections:

| Step | Background Color | Texture Overlay | CSS |
|------|-----------------|-----------------|-----|
| 1 | `#fbf8f7` (off-white) | none | `background-color: #fbf8f7` |
| 2 | `#ffffff` (pure white) | none | `background-color: #ffffff` |
| 3 | `#EBEBEB` (concrete-light) | `concrete-wall.png` | `background: #EBEBEB url(/textures/concrete-wall.png) repeat` |
| 4 | `#C1BFB2` (concrete-warm) | `asfalt-dark.png` | `background: #C1BFB2 url(/textures/asfalt-dark.png) repeat` |

Texture application: transparent PNG overlaid on base color via CSS `background-image` with `repeat`. Keep opacity subtle (D-36).

### Accent Reserved For (explicit list)
- CTA button fills ("Tickets", exhibition CTA buttons)
- Footer dark bottom bar (`#0f0f0f`)
- Active carousel progress tab fill
- Mode toggle active state indicator
- Link underlines on hover (2px bottom border per D-39)
- Hamburger menu icon strokes

### Semantic Colors via CSS Custom Properties
All components MUST use semantic tokens (`--color-surface`, `--color-text`, `--color-accent`, `--color-decorative`), never primitive values directly. This ensures mode switching works automatically.

Source: Phase 1 CONTEXT.md D-04/D-05/D-06, Phase 2 CONTEXT.md D-26/D-35, existing `src/styles/tokens/`

## Interaction States (D-14 through D-17)

### Desktop Hover
- Effect: clip-path background wipe revealing accent color
- Timing: 200ms ease-in (in), 350ms ease-out (out) -- asymmetric for premium feel
- Gate: `@media (hover: hover)` only -- prevents sticky hover on touch devices (D-15)

### Filled CTA Buttons (D-16)
- Effect: inverse wipe -- background already filled, wipe reveals contrasting color
- Timing: same asymmetric 200ms/350ms
- Easing: same as hover

### Touch / Press (D-14)
- Effect: `scale(0.96)` press pulse
- Timing: 80ms
- Global: `touch-action: manipulation` on `*` to eliminate 300ms tap delay (D-17)

### Keyboard Focus (D-14)
- Effect: `outline: 2px solid var(--color-text)` with `outline-offset: 2px`
- Already established in `base.css` `:focus-visible` rule

### Link Underlines (D-39)
- Footer links: `border-bottom: 2px solid currentColor`
- Nav links: clip-path wipe (same as general hover pattern)

## Component Inventory

### Navigation Bar (D-07 through D-13)

**Desktop (1024px+):**
- Container: floating bar, `position: fixed`, topmost z-index
- Background: `#EBEBEB` with transparent padding around element
- Layout: `display: flex; align-items: center; justify-content: space-between`
- Left: search icon (SVG magnifying glass, 24x24px, 44x44px tap target)
- Center: Kunsthaus Zurich logo with `mix-blend-mode: exclusion`
- Right: nav links ("Visit us", "Collection", "Exhibitions", "About") + "EN" language dropdown
- Typography: DINNextW1G Regular 16px for all items
- Scroll behavior (D-10): full nav dissolves on scroll, logo persists as floating island top-center. Velocity-aware: fast scroll = instant hide, slow scroll/pause = fade back with blur-expand. Search icon migrates to corner.

**Mobile (<768px, D-12):**
- Always visible bar: search icon | Kunsthaus Zurich logo (centered) | hamburger icon
- Never hidden on scroll
- Hamburger opens full-screen overlay (D-13): large DIN type, logo centered top, vertical nav links, language options below, search field at bottom

### Hero Section (D-01 through D-06)

- **Position:** fixed/pinned -- stays in place while sections scroll over (D-06)
- **Typography:** "KUNST / HAUS / ZURICH" -- right-aligned, three lines, largest type scale, topmost z-index
- **Images:** exhibition images in offset scattered arrangement, grid-aligned but not aligned to each other, stacked collage effect (D-02)
- **Carousel:** auto-plays through featured exhibitions (D-03)
- **Navigation:** labeled progress tabs -- one per exhibition with title + animated fill bar for auto-play timing (D-04). Mobile: active title visible, inactive tabs collapse to compact indicators.
- **Pause/play:** toggle for auto-play (D-05, WCAG 2.2.2)
- **CTA:** button per slide linking to exhibition detail (placeholder href)
- **Accessibility:** `<section aria-roledescription="carousel">`, each slide `role="group" aria-roledescription="slide" aria-label="X of N"`, `aria-live="polite"` (D-41)

### Sticky CTA + Mode Toggle (D-18 through D-22)

**Desktop (1024px+, D-18):**
- Two floating buttons, bottom-right corner
- "Plan a visit" / "I'm here" mode toggle (two-state switch)
- "Tickets" button (external link to kunsthaus.ch)
- Fraction counter + progress arc: bottom-left corner (D-27)

**Mobile/Tablet (D-19):**
- Sticky bottom bar: "Tickets" CTA left, mode toggle right
- Same pairing, different container

**Multilingual labels (D-22):**
- EN: "Plan a visit" / "I'm here" / "Tickets"
- DE: "Besuch planen" / "Ich bin da" / "Tickets"
- FR: "Planifier" / "Je suis ici" / "Billets"

### Section Progress Indicator (D-27)
- Position: bottom-left corner, fixed
- Display: fraction counter "03 / 07" with thin circular progress arc
- Typography: `--type-sm` (12px), `--weight-regular`
- Accessibility: `aria-label="Section X of 7: [name]"`, keyboard-focusable, `aria-live="polite"`
- Driven by GSAP ScrollTrigger callbacks

### Cinematic Reveal Strip (D-28)
- Full-bleed image masked/cropped behind colored overlay
- On scroll: mask expands via clipPath, overlay fades, title + editorial line enter from opposite sides
- Mobile: vertical wipe, text below image
- Accepts `contentType` prop and `data` array from JSON (D-30)
- Used for: featured exhibition in Exhibitions section (D-29)

### Pinned Narrative Sequence (D-28)
- Section pins in viewport, content cards scroll through one at a time
- Cards offset alternating left/right, background shifts between cards
- Mobile: unpin, cards become vertical scroll stack (ScrollTrigger matchMedia)
- Accepts `contentType` prop and `data` array from JSON (D-30)
- Used for: exhibition list, offerings, education sections (D-29)

### Exhibition Cards (D-31 through D-34)
- Image-dominant: 60vh image / 40vh text (desktop), 50/50 (mobile)
- Image aspect ratio: 3:2 landscape, `object-fit: cover` (D-32)
- Content: title, date range, description, CTA button
- Card transitions: crossfade with offset slide, alternating direction (D-33)
- Separate from hero carousel component (D-34)

### Footer (D-35 through D-39)
- Top border: 4px solid black (D-35)
- Background: white (D-35)
- Marquee: "KUNSTHAUS ZURICH" in large bold DIN, continuous CSS `animation` right-to-left, pauses on hover (D-36)
- Four-column layout (D-37):
  - Col 1: Address (Heimplatz, 8001 Zurich), Google Maps link, email, phone
  - Col 2: "Open today from 10:00" dynamic status, full weekly schedule, closing note
  - Col 3: Quick links (Exhibitions, Tickets, Membership) + Media (Press area, Press releases, Image downloads)
  - Col 4: Newsletter description + subscribe link
- Dark bottom bar `#0f0f0f` (D-38): social icons (Facebook, Instagram), legal links, copyright
- Responsive: columns stack on mobile (Claude's discretion)
- All content in EN and DE (D-43)

### Search Overlay (D-11)
- Visual shell only -- no functional search
- Opens from search icon click
- Contains: input field + hardcoded suggestion links
- Layout and styling: Claude's discretion

### Hamburger Menu Overlay (D-13)
- Full-screen overlay, dark or matching surface color
- Large DIN typography for nav items
- Kunsthaus logo centered at top
- Nav links stacked vertically
- Language options (EN / DE / FR) below nav links
- Search field at bottom

## Layout Contracts

### Grid System (from Phase 1)
- 48-track fine grid, 12 layout columns (every 4th track)
- Full-bleed, extending to viewport edges
- Responsive: desktop 48 tracks / tablet 32 / mobile 16
- CSS: existing `src/styles/grid.css` with `.col-1` through `.col-12` helpers

### Section Grid Usage (D-23)
- Each section uses the 48-track grid differently -- varied asymmetric layouts
- Hero: full-bleed
- Exhibitions: 8+4 column split
- Visit info: 3-column grid
- No uniform layout across sections

### Responsive Breakpoints
| Breakpoint | Width | Grid Columns | Section Spacing |
|------------|-------|-------------|-----------------|
| Desktop | 1024px+ | 12 (48 tracks) | 128px (`--space-32`) |
| Tablet | 768-1023px | 8 (32 tracks) | 96px (`--space-24`) |
| Mobile | <768px | 4 (16 tracks) | 64px (`--space-16`) |

### 3D Stacking Scroll Effect (D-25)
- Desktop (1024px+): pin, scale to ~90%, rotate 2-3deg (varied), fade opacity, fully covered
- Tablet (768-1023px): scale + opacity only, no rotation
- Mobile (<768px): no pinning, native-scroll fade-up (`opacity` + `translateY`)
- Reduced motion: no transforms, static layout only

### Empty States
This phase renders from JSON mock data. No empty states exist -- all sections always have content. If a JSON file fails to load at build time, Astro's build will error (Zod validation), preventing deployment with missing content.

### Error States
Not applicable for Phase 2. This is a static site -- no runtime data fetching, no form submissions, no user input. Errors are caught at build time by Zod schema validation.

### Destructive Actions
None in Phase 2. Mode toggle is non-destructive (switches `[data-mode]` attribute, no data loss).

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| WCAG level | AA (Phase 1 D-10) |
| Color contrast | 4.5:1 normal text, 3:1 large text. Concrete mid-tones never for text (D-06) |

| Tap delay | `touch-action: manipulation` globally (D-17) |
| Hover gating | `@media (hover: hover)` for all hover effects (D-15) |
