# Phase 03 Plan 03: KJM Exhibition Detail Page

**Duration estimate:** 4–5 hours
**Risk level:** Medium — eight new/modified files, one content-collection schema rewrite (with build-breaking potential), two new client-side components (video hero + IntersectionObserver slider). The CinematicReveal reuse, schema rewrite, and base-path discipline all have existing 03-02 traps that this plan must explicitly defuse.

## Goal

Deliver a full `/Kunsthaus/onsite/kerry-james-marshall` exhibition detail page that drills in from the clickable KJM card on `/onsite`. A visitor tapping that card sees, in order: a full-bleed autoplaying hero video with a top-left "← Your visit" back link → a CinematicReveal (reusing the component, with the new `imagePosition: 'left'` prop) using `kjm-portrait.png` → a dedicated intro-copy stacking section → a one-per-viewport horizontal ArtworkSlider with 10 Marshall works (each slide carrying a silent-stub headphones button for 03-04 to wire) → the "I try to make the black paints…" ArtistQuote → a HorizontalSlider of 3 video testimonials → a HorizontalSlider of 3 shop items → the existing Footer + StickyCTA. The page renders under the `data-mode="onsite"` dark cascade with the same stacking-section UX used on `/onsite/index.astro`.

This plan satisfies roadmap Phase-03 Success Criterion #4 and sets up 03-04 to wire the headphones stub into the real paywall + audio guide overlay without any rework of 03-03 components.

## Success Criteria (what must be TRUE by the end)

1. `http://localhost:4321/Kunsthaus/onsite/kerry-james-marshall` renders successfully (HTTP 200, no build errors, no runtime JS errors in the browser console).
2. The page is served under `<html data-mode="onsite">` via BaseLayout's `mode="onsite"` prop; the dark softblack palette cascades to every new component.
3. The hero video `<video>` element has all five attributes: `autoplay`, `muted`, `loop`, `playsinline`, `preload="metadata"`, plus a `poster` attribute pointing at `${base}/video/kjm-hero-poster.png`. Its `<source>` (or `src`) is `${base}/video/kjm-hero.mp4`. NO `transition:persist` attribute is present.
4. A top-left back link overlay reads "← Your visit" and links to `${base}/onsite`. It is `position: absolute` inside the hero, visually above the video, with a legible contrast.
5. Immediately after the hero, a `CinematicReveal` renders inside a `stacking-section cinematic-reveal-section` wrapper (matching the onsite/index.astro pattern). It uses `kjm-portrait.png` as the image. The page builds a `cinematicData` object that maps `cardTitle → title` (NOT passing `cardTitle` as a direct prop — that trap bit 03-02).
6. `CinematicReveal.astro` accepts a new optional `imagePosition` prop (`'left' | 'center' | 'right'`, default `'center'`). The KJM page passes `'left'`. On desktop (≥768px) the cinematic reveal image shifts horizontally via a CSS custom property `--cinematic-image-align`. Mobile is unchanged (centered layout).
7. An intro-copy SectionWrapper renders between the cinematic reveal and the ArtworkSlider, containing `intro.heading` as the SectionWrapper `sectionName` and `intro.body` as its content slot.
8. The ArtworkSlider SectionWrapper contains one `<ArtworkSlider>` rendering exactly 10 slides, each with: number, title (EN), year, collection, optional credit, description (EN), cover image, and a headphones `<button>` styled as an icon button. Prev/next buttons scroll the track by one slide (`scrollBy({ left: slideWidth, behavior: 'smooth' })`). The slider has `role="region"` and `aria-roledescription="carousel"`; each slide is a list item.
9. The headphones button on each artwork is a **silent stub**: it has `type="button"`, a recognizable icon (`🎧` or Material icon `headphones`), an `aria-label="Play audio guide for {title}"`, and on click does NOTHING (empty click handler or no onclick attribute at all). It does NOT dispatch any custom event, does NOT call `window.__audioGuideOverlay`, does NOT log to the console. 03-04 will wire it up.
10. The ArtistQuote component renders the Marshall quote «I try to make the black paints that I use as complex as any other colour on the palette.» with an attribution line "— Kerry James Marshall". Rendered in its own SectionWrapper.
11. A HorizontalSlider (variant: videos) renders 3 video testimonial cards, each with a thumbnail image, a duration label (e.g. "1 Min"), a quote, and a role label. Rendered in its own SectionWrapper.
12. A HorizontalSlider (variant: shop) renders 3 shop item cards, each with a product image, a title, a price (e.g. "CHF 750.-"), and a non-functional "Add to cart" button. Rendered in its own SectionWrapper.
13. HorizontalSlider uses CSS `scroll-snap-type: x mandatory` with `scroll-snap-align: center` on each card. An IntersectionObserver maps intersection ratio → CSS custom properties `--card-scale` and `--card-opacity` on each card. Under `prefers-reduced-motion: reduce`, the observer short-circuits to full scale/opacity and no writes fire during scroll.
14. `src/content.config.ts` has its `onsiteExhibition` collection REWRITTEN to the new schema: `{ id, title, dateLabel, heroVideo, cinematicReveal, intro, artworks[], artistQuote, videos[], shop[] }`. The old fields (`practicalInfo`, `audioGuideAvailable`, `artworks[].room`, `artworks[].medium`, `artworks[].audioGuide`, `artworks[].membersOnly`) are removed. Build passes (content sync succeeds with the new schema validation).
15. `src/content/onsite/kerry-james-marshall.json` is REWRITTEN. Top-level structure is `{ "kerry-james-marshall": { id: "kerry-james-marshall", ... } }` (id-keyed object wrapper).    ```astro
   ---
   interface Props { /* as above */ }
   const { title, dateLabel, video, backLabel, backHref, lang } = Astro.props;

     </div>
   </section>

   <script>
     // Reduce-motion: pause the video.
     function kjmHeroReduceMotionGuard() {
       if (!window.matchMedia('(prefers-reduced-motion: reduce)').matches) return;

     }
     kjmHeroReduceMotionGuard();
     document.addEventListener('astro:page-load', kjmHeroReduceMotionGuard);
